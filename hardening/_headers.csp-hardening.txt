# ============================================================================
# CSP HARDENING — closes gap R2 (script-src 'unsafe-inline') SAFELY.
#
# WHY report-only first: the platform is inline-script-heavy. Flipping the
# enforced CSP to remove 'unsafe-inline' blind WOULD break inline handlers
# (regression risk = HIGH). Instead, ship a *Report-Only* tightened policy
# ALONGSIDE the current enforced one. Browsers execute everything as today but
# report what WOULD have been blocked, so you find every inline script before
# enforcing. Zero user-facing risk.
#
# ROLLOUT:
#   1. Add the Report-Only line below to frontend/_headers under the /* block.
#   2. Add a /api/csp-report endpoint (or use report-uri.com) to collect reports.
#   3. Migrate flagged inline scripts to external files or add per-script nonces.
#   4. When reports are clean for ~1 week, move the tightened policy into the
#      enforced Content-Security-Policy line and delete 'unsafe-inline'.
#   5. Validate with hardening/handlers-csp note + the header-assert CI job.
#
# ROLLBACK: delete the Report-Only line. It never blocks anything, so rollback
# is risk-free even mid-rollout.
# ============================================================================
