Integration Playbook

Get Production-Connected in 5 Steps

From zero to fully integrated with your SIEM, TIP, and identity stack.

1

Enterprise SSO — Azure AD / Okta

Register CYBERDUDEBIVASH as an OIDC app in your Azure AD or Okta. Set the Redirect URI, then call POST /api/auth/enterprise/configure with your tenant ID and client credentials. Your entire team can then authenticate at /api/auth/enterprise/sso?org=your-org using their corporate identity.

GET /api/auth/enterprise/config POST /api/auth/enterprise/configure GET /api/auth/enterprise/sso?org=<slug>
2

SIEM Integration — Splunk, Sentinel, QRadar, Elastic

Register your SIEM endpoint once. After that, every new critical CVE can push a production-ready detection rule (Sigma/KQL/SPL/YARA) directly to your SIEM via the platform's rule deployment engine.

POST /api/integrations/configure POST /api/integrations/test POST /api/integrations/deploy GET /api/integrations/deploy-log
3

Threat Intelligence Pull — TAXII 2.1

Configure your Threat Intelligence Platform (Cisco ThreatResponse, Oracle CASB, MISP, OpenCTI, CrowdStrike, Anomali) to pull from our TAXII 2.1 server. CVE feed is free; IOC and actor feeds require PRO/ENTERPRISE.

GET /api/taxii/discovery GET /api/taxii/collections GET /api/taxii/collections/cve-feed/objects GET /api/taxii/collections/kev-feed/objects
4

Machine-to-Machine API Keys

Generate scoped API keys for your CI/CD pipelines, SOAR playbooks, and automated workflows. Per-key rate limiting, usage metering, and self-serve rotation.

POST /api/keys/generate GET /api/keys/usage GET /api/docs
5

Automate with Workflows

Define trigger-action workflows: when a new CRITICAL CVE arrives → deploy detection rule to Splunk → create SOC case → push PagerDuty alert. All persisted in D1, auditable, and recoverable.

POST /api/workflows POST /api/workflows/:id/execute GET /api/audit/log
